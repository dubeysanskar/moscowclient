## 🐛 Issues Found & Fixed

### Canonical URL Bug (Critical — Fixed Session 13)
**Problem:** The root `layout.js` set `alternates.canonical: "https://tahaairwaves.ru"` which cascaded to ALL child pages. Every page on the site had its canonical pointing to the homepage, making search engines think every page was a duplicate of the homepage.

**Impact:** Search engines would ignore 77 out of 78 pages, indexing only the homepage.

**Fix:** Added per-page `alternates.canonical` to every page and layout that overrides the parent's canonical:
- Static pages: explicit `metadata.alternates.canonical` in each `page.js`
- Dynamic routes: `generateMetadata()` returns page-specific canonical
- SEO landing pages: `alternates.canonical` added to each `layout.js`

**Lesson:** Never set a hardcoded canonical in the root layout. Either make it dynamic based on the current path, or set canonicals only at the page level.
